# broken-app

Учебный проект: поиск и исправление ошибок, оптимизация производительности.

## Использованные инструменты

| Инструмент | Что проверяет |
|------------|---------------|
| `cargo test` | Корректность логики |
| `cargo +nightly miri test` | Undefined Behavior (выходы за границы, use-after-free) |
| `valgrind -leak-check=full --show-leak-kinds=all` | Утечки памяти |
| ASan (`-Zsanitizer=address`) | Ошибки работы с памятью |
| TSan (`-Zsanitizer=thread`) | Гонки данных |
| `perf record/report` + `cargo flamegraph` | Профилирование, поиск узких мест |
| `cargo bench` (baseline + criterion) | Замеры производительности до/после |

## Найденные и исправленные баги

### 1. Выход за границу среза в `sum_even`

Цикл `0..=values.len()` итерировался на один элемент дальше конца среза. В сочетании с `get_unchecked` — Undefined Behavior. Miri: `'assume' called with 'false'`.

**Исправление:** unsafe удалён, функция заменена на безопасный итератор.

### 2. Use-after-free в `use_after_free`

Чтение указателя после `Box::from_raw` и освобождения памяти.

**Исправление:** функция удалена.

### 3. Утечка памяти в `leak_buffer`

Память через `Box::into_raw` не освобождалась. Valgrind: `definitely lost: 5 bytes`.

**Исправление:** unsafe удалён, обход без аллокаций.

### 4. Неверная логика `average_positive`

Усреднение по всем элементам, а не только по положительным. Для `[-5, 5, 15]` выдавал 5 вместо 10.

**Исправление:** добавлен фильтр `> 0`.

### 5. Неполная обработка пробелов в `normalize`

`replace(' ', "")` удалял только пробелы, игнорируя табуляции и переносы строк.

**Исправление:** замена на `split_whitespace()`.

### 6. Гонка данных в `race_increment`

`static mut` + `+= 1` из нескольких потоков без синхронизации.

**Исправление:** замена на `AtomicU64` с `fetch_add`.

## Оптимизация

### `slow_fib`: O(2ⁿ) → O(n)

Рекурсия заменена на итеративный проход. Ускорение в **~346 000 раз** (с 5.83 мс до 16.9 нс по criterion). До оптимизации функция занимала 73.7% всего времени (perf), после — не видна в профилировщике.

### `slow_dedup`: O(n²) → O(n log n)

Линейный поиск дубликатов и сортировка на каждой вставке заменены на `HashSet` и одну сортировку в конце. Ускорение в **~93 раза** (с 9.67 мс до 104.5 мкс по criterion).

## Регрессионные тесты
